# thumbnails

**Purpose:** Standalone CLI tool that generates 5 branded YouTube thumbnail candidates (concept brainstorm → AI base image → Pillow composite) for a First Cup episode title/description. **STATUS: work-in-progress, uncommitted, and NOT wired into the main pipeline.** `thumbnail_generator.py`, `branding/`, and `THUMBNAIL_PLAN.md` are all untracked in git; `requirements.txt` has uncommitted additions of `google-genai` and `Pillow`. Grep confirms no other file (`youtube_processor.py`, `slack_helper.py`, etc.) imports `thumbnail_generator` — it only runs via its own `if __name__ == "__main__"` CLI entry point, invoked manually.

## Key files
- `thumbnail_generator.py` → `get_visual_concepts`, `get_default_concepts`, `generate_base_image`, `create_gradient_background`, `composite_thumbnail`, `add_branding_overlay`, `generate_thumbnails`, `interactive_refinement`, `main` — full single-file implementation (809 lines).
- `THUMBNAIL_PLAN.md` → design doc / research notes written before implementation; documents intended architecture, cost estimates, and open questions (some phases in the plan, e.g. Slack interactive selection, are only partially realized in code).
- `branding/first_cup_header.png` → the only branding asset present; a ~841x104px "PRODUCT COFFEE | THE FIRST CUP" header PNG with alpha channel, composited onto generated thumbnails via `add_branding_overlay`.
- `slack_helper.py` → `SlackHelper.send_thumbnail_options` — already exists and is called from `thumbnail_generator.main()` when `--slack` is passed; this is the one point of contact with the rest of the codebase, but it's opt-in via CLI flag, not automatic.

## Key flows
- CLI entry (`main()`) → parses `title`/`description` args (or `--test` sample data) → `generate_thumbnails(title, description, ...)`.
- `generate_thumbnails`: Step 1 `get_visual_concepts` (Claude call, falls back to `get_default_concepts` if no `ANTHROPIC_API_KEY` or on error) → for each of 5 concepts: Step 2 `generate_base_image` (Gemini call, falls back to `create_gradient_background` on any failure/missing key) → Step 3 `composite_thumbnail` (style-based enhance/tint + left-side gradient overlay + headline text with auto-shrinking font + `add_branding_overlay`) → saves PNG to `outputs/thumbnails_{safe_title}_{timestamp}/thumbnail_{i}_{style}.png` plus a `concepts.json` sidecar.
- Post-generation: if `--slack` and Slack configured → `SlackHelper.send_thumbnail_options`; else (and unless `--no-feedback`) → `interactive_refinement` CLI loop (regenerate one/all, edit headline, give free-text feedback that gets appended to the Gemini prompt).

## Mechanism notes  ← HIGHEST SIGNAL
- **Dual-API design, both optional/soft-fail:** Claude (`claude-sonnet-5`, `thinking: {"type": "disabled"}`, 2000 max_tokens) brainstorms 5 concepts as JSON; Gemini (`gemini-2.0-flash-exp-image-generation`, note: THUMBNAIL_PLAN.md's research targeted `gemini-2.5-flash-image` but the shipped code calls the older `-exp-` model name) generates the base image per concept. Both wrapped in bare `try/except Exception` that logs and silently falls back (to `get_default_concepts` / a flat gradient background) rather than raising — a generation run always "succeeds" even with zero AI calls, which can silently produce low-quality output with no error signal to the caller.
- **Key/quota coupling called out by CLAUDE.md:** `GOOGLE_API_KEY` is documented (project CLAUDE.md / `.env`) as the *same* key already used for `YOUTUBE_API_KEY`, just with the Generative Language API additionally enabled on that Google Cloud project — so exhausting/revoking this key has blast radius on the existing YouTube Data API integration (`blog_publisher.py`'s video/thumbnail lookups), not just image generation.
- **File overwrite risk:** each `generate_thumbnails()` call makes a fresh timestamped directory (`thumbnails_{safe_title}_{timestamp}`), so back-to-back runs don't collide. But within `interactive_refinement`'s single-thumbnail regenerate path (`h N` command), the old file is explicitly deleted (`saved_paths[idx].unlink(missing_ok=True)`) and replaced with a `_v2` suffixed name — the free-text feedback path instead writes a new `_v{HHMMSS}` file without deleting the original, so that branch accumulates versions rather than overwriting.
- **No retry logic anywhere** — every API failure (Claude or Gemini) is a single attempt with immediate fallback, no backoff/retry, unlike `youtube_processor.py`'s handling of `APICreditsExhaustedError`/`APIRateLimitError`. A Gemini quota/credits exhaustion here just silently degrades every thumbnail to a flat gradient background with no pause-and-resume behavior and no Slack alert.
- **Cost per generation:** THUMBNAIL_PLAN.md estimates ~$0.039/image × 5 = ~$0.20/episode for Gemini image generation, on top of a small Claude call for concepts; combined with existing Claude processing (~$0.15-0.30), plan estimates ~$0.35-0.50/episode total if this were wired in.
- **Branding dependency is minimal and fragile:** only one asset (`branding/first_cup_header.png`) is used; `add_branding_overlay` hardcodes scaling to 45% of thumbnail width and randomly picks a corner (excluding bottom-right, reserved for YouTube's timestamp overlay) unless a position is passed. No brand color/font asset files exist in `branding/` — `BRAND_COLORS` and `FONT_PATHS` are hardcoded constants inside `thumbnail_generator.py` itself, and `FONT_PATHS` points at font files under a sibling project directory (`~/developer/productcoffee/session-thumbnail-generator/fonts/...`) that may not exist on other machines, falling back to system Arial/Helvetica or PIL's default bitmap font if none of those paths resolve.
- **Integration status:** no config.json flag, no call site in `youtube_processor.py`'s pipeline, no Launch Agent wiring. THUMBNAIL_PLAN.md's "Phase 2: Pipeline Integration" and "Phase 3: Slack Integration" checklists are unchecked; the Slack piece is partially done in code (`send_thumbnail_options` exists) but nothing triggers it automatically — it only fires if a human runs the script by hand with `--slack`.

## Related subsystems
- **youtube_processor.py** (main pipeline) — not connected; would be the natural integration point per THUMBNAIL_PLAN.md Option A (auto-generate after title selection), currently unimplemented.
- **slack_helper.py** — one-way dependency: `thumbnail_generator.py` optionally calls `SlackHelper.send_thumbnail_options`; no reverse dependency.
- **blog_publisher.py / youtube_helper.py** — shares the same `GOOGLE_API_KEY` credential (Generative Language API vs. YouTube Data API scopes on one key), so credential rotation or quota issues can cross-affect both subsystems even though they are code-independent.
