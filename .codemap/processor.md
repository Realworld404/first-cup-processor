# processor

**Purpose:** Watches `transcripts/` for new First Cup panel transcripts and runs a 4-step Claude pipeline (title options ‚Üí YouTube description/keywords ‚Üí newsletter teaser ‚Üí blog post), saving outputs per-episode and handing off to Slack/publish-poller for review and WordPress publishing.

## Key files
- `youtube_processor.py` ‚Üí `watch_directory`, `process_transcript_file`, `process_with_claude_pipeline`, `call_claude_api`, `interactive_title_selection` ‚Äî main loop, per-file orchestration, the pipeline itself, the shared API-call wrapper, and blocking title selection.
- `youtube_processor.py` ‚Üí `create_youtube_description_prompt`, `create_newsletter_teaser_prompt`, `create_blog_post_prompt`, `get_titles_from_claude` ‚Äî Step 2/3/4/1 prompt builders (focused prompts, not the deprecated mega-prompt).
- `youtube_processor.py` ‚Üí `parse_youtube_description_response`, `parse_newsletter_teaser_response`, `parse_blog_post_response` ‚Äî regex-based extractors for Step 2/3/4, coupled to the prompt's section headers (`HOOK:`, `KEY_TOPICS:`, `TIMESTAMPS:`, `PANELISTS:`, `KEYWORDS:`, `NEWSLETTER TEASER:`, `LINKEDIN/BLOG POST:`).
- `youtube_processor.py` ‚Üí `create_prompt`, `process_with_claude`, `parse_response` ‚Äî DEPRECATED single-mega-prompt path, kept only for reference; not on the live call path from `process_transcript_file`.
- `youtube_processor.py` ‚Üí `APICreditsExhaustedError`, `APIRateLimitError`, `_credits_paused_for_file` ‚Äî error taxonomy and the module-level pause flag (see Mechanism notes).
- `youtube_processor.py` ‚Üí `save_outputs`, `populate_template`, `load_template` ‚Äî writes `SELECTED_TITLE.txt`, `youtube_description.txt`, `keywords.txt`, `newsletter_teaser.txt`, `linkedin_blog_post.txt`, `description_components.txt`, `full_response.txt` into a per-episode output dir.
- `youtube_processor.py` ‚Üí `spawn_publish_poller` ‚Äî fires `publish_poller.py` as a detached subprocess after a successful save.
- `youtube_processor.py` ‚Üí `load_processed_files`, `save_processed_file`, `PROCESSED_FILE` (`.processed_transcripts.json`) ‚Äî dedup/skip bookkeeping for the watch loop.
- `model_registry.py` ‚Üí `get_model()` ‚Äî single source of truth for the Anthropic model id (reads `config.json` `api.model`, default `claude-sonnet-5`; `ANTHROPIC_MODEL` overrides). Every `call_claude_api` invocation calls it; no model literal is hardcoded at a call site. `set_model()` persists a new choice; `ModelUnavailableError` + `suggest_alternatives()` drive the deprecation-recovery flow.
- `test_parse_response.py` ‚Üí tests `parse_response` (the DEPRECATED mega-prompt parser), NOT the three parsers actually used in production (`parse_youtube_description_response`/`parse_newsletter_teaser_response`/`parse_blog_post_response`). Coverage gap ‚Äî see Mechanism notes.
- `test_pipeline.py` ‚Üí integration smoke test that calls `process_with_claude_pipeline` directly against a real transcript file and asserts every output field is non-empty; makes live Anthropic API calls (costs money, needs `ANTHROPIC_API_KEY`).

## Key flows
- **Watch loop:** `main()` ‚Üí `watch_directory()` ‚Üí globs `transcripts/*.{txt,md,json}` every `WATCH_INTERVAL` (10s) ‚Üí skips names already in `.processed_transcripts.json` or with a `FAILED_` counterpart ‚Üí `process_transcript_file()` per new file.
- **Per-file pipeline:** `process_transcript_file()` ‚Üí `interactive_title_selection()` (Step 1, BLOCKS) ‚Üí `process_with_claude_pipeline()` runs Step 2 (`create_youtube_description_prompt` ‚Üí `call_claude_api` ‚Üí `parse_youtube_description_response`) ‚Üí 30s sleep ‚Üí Step 3 (`create_newsletter_teaser_prompt` using Step 2's `hook` ‚Üí `call_claude_api` ‚Üí `parse_newsletter_teaser_response`) ‚Üí 30s sleep ‚Üí Step 4 (`create_blog_post_prompt` ‚Üí `call_claude_api` ‚Üí `parse_blog_post_response`) ‚Üí `load_template`/`populate_template` ‚Üí `save_outputs` ‚Üí `save_processed_file` ‚Üí Slack completion notice ‚Üí `spawn_publish_poller`.
- **Manual/test entry points:** `main()` also handles `--test-slack` (calls `SlackHelper.test_connection`, no pipeline) and `-h/--help`; non-watch "manual mode" is really just the same `watch_directory` call driven by the Launch Agent dropping a file once.
- **Credits-exhaustion recovery:** `call_claude_api` raises `APICreditsExhaustedError` ‚Üí caught in `process_transcript_file` ‚Üí with Slack, `slack.notify_credits_exhausted()` + blocking `slack.poll_for_resume()` then clears `_credits_paused_for_file` and returns `None` (file retried next loop, never marked FAILED_); without Slack, sets `_credits_paused_for_file` and returns `None`, and `watch_directory`'s loop checks that flag every iteration to block with billing instructions until the process is restarted.
- **Rate-limit path (deliberately different):** `APIRateLimitError` ‚Üí `slack.notify_error()` (if enabled) ‚Üí `save_processed_file(output_dir, f"FAILED_{filename}")` ‚Äî permanent skip, no auto-retry, by design (rate limits resolve on their own; user manually re-queues).

## Mechanism notes  ‚Üê HIGHEST SIGNAL
- **Shared mutable state across call paths:** `_credits_paused_for_file` is a bare module-level global written from inside `process_transcript_file` (called once per file) and read from inside `watch_directory`'s `while True` loop (a different call frame, same process). It only matters when Slack is unavailable ‚Äî with Slack, the block happens synchronously inside `slack.poll_for_resume()` and the flag is cleared immediately after. This is the kind of implicit coordination that's easy to break by adding a second call site or by running two watcher processes against the same output dir (no file lock ‚Äî `.processed_transcripts.json` read/write is not atomic either, see `load_processed_files`/`save_processed_file`).
- **Stale-model outage already happened here.** `MODEL = 'claude-sonnet-5'` (top of file) is the one line every `call_claude_api()` call depends on. Commit `11e2a92` ("fix: migrate off retired claude-sonnet-4 model (404 crash loop)") fixed a prior outage where a retired model id caused every API call to 404 ‚Äî this is a single point of failure with no fallback/retry-on-different-model logic. When Anthropic retires a model, this line breaks the whole pipeline immediately (every step, every file).
- **`thinking={"type": "disabled"}`** is passed on every `messages.create` call in `call_claude_api`. If a future edit removes this or a model requires thinking tokens, thinking tokens would eat into the tight per-step `max_tokens` budgets (Step 3 newsletter teaser is capped at only 1000 tokens) and could silently truncate output ‚Äî watch for this if newsletter/blog posts start coming back cut off.
- **Blocking title selection stalls the entire watcher.** `interactive_title_selection()` ‚Üí `interactive_title_selection_cli()` uses a synchronous `input()` call, or `interactive_title_selection_slack()` calls `slack.poll_for_response(message_ts, timeout_seconds=None)` ‚Äî an **indefinite** blocking poll. Either way, `watch_directory`'s single-threaded loop cannot process any other dropped transcript, check for publish commands, or notice new files until the human responds. One forgotten Slack thread = the whole system appears hung.
- **Rate-limit self-throttling is a fixed sleep, not adaptive.** `process_with_claude_pipeline` hardcodes `RATE_LIMIT_DELAY = 30` seconds between Step 2‚Üí3 and Step 3‚Üí4, purely to stay under a 30k-tokens/minute tier limit. This is a blind wait, not based on actual token usage or API response headers ‚Äî if the tier limit changes or a step's prompt grows, 30s may no longer be enough and `APIRateLimitError` will start firing (which marks the file permanently FAILED_, see below).
- **Regex parsing is tightly coupled to prompt section headers.** `parse_youtube_description_response`/`parse_newsletter_teaser_response`/`parse_blog_post_response` all use lookahead regexes anchored on literal headers (`HOOK:`, `KEY_TOPICS:`, `TIMESTAMPS:`, `PANELISTS:`, `KEYWORDS:`, `NEWSLETTER TEASER:`, `LINKEDIN/BLOG POST:` or `BLOG POST:`) defined in the matching `create_*_prompt` functions. Edit a header string in the prompt without updating the regex (or vice versa) and that field silently comes back empty ‚Äî each parser does log a `‚öÝÔ∏è WARNING` but nothing hard-fails the pipeline.
- **Test coverage gap:** `test_parse_response.py` (the "regression test suite") exercises only the DEPRECATED `parse_response()` mega-prompt parser. The three parsers actually used in production (`parse_youtube_description_response`, `parse_newsletter_teaser_response`, `parse_blog_post_response`) have zero unit-level regression coverage ‚Äî only the end-to-end, API-cost-incurring `test_pipeline.py` smoke test touches them, and it asserts non-emptiness, not correctness of the regex boundaries.
- **`APIRateLimitError` vs `APICreditsExhaustedError` are distinguished by string-sniffing the exception message** (`"credit"`/`"billing"`/`"payment"` substrings in `anthropic.RateLimitError`'s text) inside `call_claude_api`. If Anthropic changes their error message wording, a real credits-exhaustion event could get misclassified as a plain rate limit and permanently FAILED_ the file instead of pausing for resume (or vice versa).
- **Fan-out/subprocess:** `spawn_publish_poller` launches `publish_poller.py` via `subprocess.Popen(..., start_new_session=True)` ‚Äî fully detached from the parent process, communicating only through `.publish_poller_state.json` (written by `slack.save_poller_state`). No supervision: if the poller crashes, the main watcher has no visibility into it.
- **Output writes are all per-episode, path-derived from `filepath.stem` + a timestamp** (`save_outputs`), so re-processing the same transcript name twice never collides ‚Äî but also means a file processed twice (e.g., manually removed from `.processed_transcripts.json`) produces a fresh sibling directory rather than overwriting.
